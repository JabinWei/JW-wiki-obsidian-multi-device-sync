# 哈希值底层原理

## 💡 核心概念

哈希函数是一种**单向数学变换**，将任意长度的输入（消息）映射为固定长度的输出（哈希值/摘要），核心特征：

- **确定性**：同一输入永远产生同一输出
- **单向性**（抗原像性）：给定哈希值，无法反推出原始输入
- **雪崩效应**：输入微小变化导致输出完全不同
- **抗碰撞性**：无法找到两个不同输入产生相同哈希值

```
输入: "Hello, World!"  → SHA-256 →  dffd6021bb2bd5b0af676290809ec3a53191dd81c7f70a4b28688a362182986f
输入: "Hello, World"   → SHA-256 →  03675ac53ed9f8e1fb3d5fad9fa85e71edda54423d618ad200e6eb45fac61a8c
```

> 🔑 核心原理：**压缩函数 + 迭代结构 + 混乱扩散**，将无限消息映射到有限摘要，通过精心设计的位运算（旋转、异或、替换）实现不可逆性。

---

## 🔍 详细说明

### 一、信息压缩与鸽巢原理

哈希函数本质是**不可逆的信息压缩**。把无限可能的消息空间映射到有限的输出空间，必然存在碰撞（鸽巢原理）：

```
输入空间: ∞ (任意长度)  →  输出空间: 2^n (n 位哈希)

SHA-256: 2^256 ≈ 1.15 × 10^77（比可观测宇宙中的原子数还多）
```

### 二、三大安全属性

| 属性 | 含义 | 攻击难度 |
|------|------|----------|
| **抗原像性** (Preimage Resistance) | 给定哈希值 h，无法找到 m 使 hash(m)=h | O(2^n) |
| **抗第二原像性** (Second Preimage Resistance) | 给定 m1，无法找到 m2 使 hash(m1)=hash(m2) | O(2^n) |
| **抗碰撞性** (Collision Resistance) | 无法找到任意 m1≠m2 使 hash(m1)=hash(m2) | O(2^{n/2})（生日攻击） |

> ⚠️ 生日攻击使抗碰撞性的安全强度只有输出位数的一半（2^{n/2} 而非 2^n）。

### 三、Merkle-Damgård 结构（MD5, SHA-1, SHA-256）

最经典的哈希构造方式，将消息分块后链式迭代：

```
消息 padding → [块1] [块2] [块3] ... [块N]
                  ↓     ↓     ↓         ↓
IV(初始向量) → 压缩函数 → 压缩函数 → 压缩函数 → ... → 最终哈希值
```

**工作流程**：

1. **填充（Padding）**：将消息填充到块大小的整数倍，末尾附加原始长度（抵抗长度扩展攻击的基础）
2. **分块处理**：每个固定大小的块依次通过压缩函数
3. **链式迭代**：上一块的输出（链接变量）作为下一块的输入
4. **最终输出**：最后一个压缩函数输出即为哈希值

### 四、压缩函数内部（以 SHA-256 为例）

SHA-256 每轮操作包含消息扩展和 64 轮迭代：

**逻辑函数定义**：

```
Ch(x,y,z)   = (x ∧ y) ⊕ (¬x ∧ z)              // 选择函数
Maj(x,y,z)  = (x ∧ y) ⊕ (x ∧ z) ⊕ (y ∧ z)     // 多数函数
Σ0(x)       = ROTR²(x) ⊕ ROTR¹³(x) ⊕ ROTR²²(x) // 大 Sigma
Σ1(x)       = ROTR⁶(x) ⊕ ROTR¹¹(x) ⊕ ROTR²⁵(x)
σ0(x)       = ROTR⁷(x) ⊕ ROTR¹⁸(x) ⊕ SHR³(x)  // 小 sigma
σ1(x)       = ROTR¹⁷(x) ⊕ ROTR¹⁹(x) ⊕ SHR¹⁰(x)
```

**核心轮函数**：

```
T1 = h + Σ1(e) + Ch(e,f,g) + K[t] + W[t]
T2 = Σ0(a) + Maj(a,b,c)
h = g,  g = f,  f = e,  e = d + T1
d = c,  c = b,  b = a,  a = T1 + T2
```

- `K[t]`：预定义的 64 个轮常量
- `W[t]`：由消息块扩展出的 64 个 32 位字
- 每轮更新 8 个工作变量（a 到 h），经过 64 轮后与初始 IV 累加

### 五、海绵结构（SHA-3 / Keccak）

更新的设计范式，天然抗长度扩展攻击，分为两个阶段：

```
吸收阶段（Absorb）：
输入 → [块] → XOR → [f] → XOR → [f] → ...
            ↑        ↑       ↑
         初始状态   变换    变换

挤压阶段（Squeeze）：
      ... → [f] → 输出块1 → [f] → 输出块2 → ...
              ↑                ↑
           变换              变换
```

- **f** 是核心置换函数 Keccak-f[1600]，对 1600 位内部状态操作
- 与 Merkle-Damgård 相比，天然抗长度扩展攻击，安全性证明更优雅

## 📎 例子

### 各语言中的哈希计算

**Python**：

```python
import hashlib

# SHA-256
h = hashlib.sha256(b"Hello, World!")
print(h.hexdigest())  # dffd6021bb2bd5b0...

# MD5（仅用于非安全场景）
h = hashlib.md5(b"Hello")
print(h.hexdigest())

# SHA-3
h = hashlib.sha3_256(b"Hello")
print(h.hexdigest())
```

**Java**：

```java
import java.security.MessageDigest;

MessageDigest md = MessageDigest.getInstance("SHA-256");
byte[] hash = md.digest("Hello, World!".getBytes(StandardCharsets.UTF_8));
// 转十六进制
StringBuilder sb = new StringBuilder();
for (byte b : hash) sb.append(String.format("%02x", b));
System.out.println(sb.toString());
```

**JavaScript (Node.js)**：

```javascript
const crypto = require('crypto');

const hash = crypto.createHash('sha256')
    .update('Hello, World!')
    .digest('hex');
console.log(hash);
```

### 哈希表索引计算

```python
# 哈希表的关键操作：index = hash(key) % table_size
key = "username"
table_size = 16
index = hash(key) % table_size
print(index)  # 确定该键存在哪个槽位
```

**负载因子** = 元素数量 / 表大小。过高时触发 **rehash**（扩容并重新散列所有元素）。

冲突解决策略：

| 策略 | 原理 | 代表 |
|------|------|------|
| **拉链法** | 每个槽位维护一个链表/红黑树 | Java HashMap |
| **开放寻址法** | 冲突时按探测序列找下一个空位 | Python dict |

### 数据完整性校验

```
下载文件 → 计算 SHA-256 → 与官方公布的哈希值比对 → 一致则完整
```

### 密码存储（正确姿势）

```python
import bcrypt

# 注册：哈希 + 随机盐存储
password = b"user_password"
salt = bcrypt.gensalt()           # 自动生成随机盐
hashed = bcrypt.hashpw(password, salt)
# 数据库中存储 hashed

# 登录验证
bcrypt.checkpw(password, hashed)  # True / False
```

> ❌ 绝不要用 SHA-256 / MD5 直接存储密码（彩虹表攻击）。  
> ✅ 使用 bcrypt / argon2 / scrypt 等专用密码哈希函数。

## 🔬 变量可哈希的概念

### 什么是「可哈希」（Hashable）？

**可哈希**是指一个对象在其生命周期内拥有**不变的哈希值**，并且可以与其他对象进行**相等比较**。这是对象能否作为字典键（`dict` key）或集合元素（`set` element）的根本前提。

### 哈希协议：`__hash__` 与 `__eq__`

Python 中，`hash()` 函数调用对象的 `__hash__()` 方法。一个对象要成为可哈希的，必须满足：

```
1.  定义了 __hash__() 方法（返回整数哈希值）
2.  定义了 __eq__() 方法（用于比较相等）
3.  如果 a == b，则必须 hash(a) == hash(b)  ← 核心契约
4.  哈希值在对象生命周期内不变
```

### 为什么要求「哈希值不变」？

哈希表（Python 字典、集合）依靠哈希值定位槽位：

```
插入：槽位 = hash(key) % table_size  →  把 value 存入该槽位
查找：同样的公式  →  定位到同一个槽位  →  取出 value
```

如果哈希值中途改变，之前存储的槽位就**找不到**了——这是致命的。

### 可变 vs 不可变：可哈希性判断

| 类型 | 可变？ | 可哈希？ | 可作为 dict 键？ |
|------|--------|----------|-------------------|
| `int`, `float`, `str`, `bytes` | ❌ 不可变 | ✅ 可哈希 | ✅ |
| `tuple`（元素全可哈希） | ❌ 不可变 | ✅ 可哈希 | ✅ |
| `frozenset` | ❌ 不可变 | ✅ 可哈希 | ✅ |
| `list` | ✅ 可变 | ❌ 不可哈希 | ❌ |
| `dict` | ✅ 可变 | ❌ 不可哈希 | ❌ |
| `set` | ✅ 可变 | ❌ 不可哈希 | ❌ |
| `tuple`（包含 list） | ❌ 不可变 | ❌ 不可哈希 | ❌ |
| 自定义类（默认） | ✅ | ✅（基于 id） | ✅ |

### 代码验证

```python
# ✅ 不可变类型 → 可哈希
print(hash("hello"))       # 稳定的哈希值
print(hash((1, 2, 3)))     # 元组可哈希
print(hash(frozenset([1,2,3])))  # 冻结集合可哈希

# ❌ 可变类型 → 不可哈希
try:
    hash([1, 2, 3])        # TypeError: unhashable type: 'list'
except TypeError as e:
    print(e)

try:
    hash({"key": "value"}) # TypeError: unhashable type: 'dict'
except TypeError as e:
    print(e)

# ❌ 包含可变元素 → 也不可哈希
t = (1, [2, 3])  # 元组本身不可变，但包含可变 list
try:
    hash(t)        # TypeError: unhashable type: 'list'
except TypeError as e:
    print(e)
```

### 自定义类的可哈希性

```python
# 默认行为：基于对象身份 (id) 的哈希
class Person:
    def __init__(self, name):
        self.name = name

p1 = Person("Alice")
p2 = Person("Alice")
print(hash(p1) == hash(p2))  # False（不同对象，不同哈希）
print(p1 == p2)               # False（默认 __eq__ 比较 id）

# 自定义：基于值的哈希和相等
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __eq__(self, other):
        if not isinstance(other, Person):
            return NotImplemented
        return self.name == other.name and self.age == other.age

    def __hash__(self):
        # 基于不变属性的哈希
        return hash((self.name, self.age))

p1 = Person("Alice", 30)
p2 = Person("Alice", 30)
print(hash(p1) == hash(p2))  # True（值相同，哈希相同）
print(p1 == p2)               # True

# 现在可以作为集合元素和字典键
people = {p1, p2}   # 自动去重
print(len(people))  # 1
```

> ⚠️ **关键规则**：如果自定义 `__eq__` 而不定义 `__hash__`，Python 会自动将 `__hash__` 设为 `None`，使对象不可哈希。如果 `__eq__` 基于可变属性，那么不应该定义 `__hash__`——否则哈希值可能改变，破坏哈希表契约。

## ⚡ 反例 / 边界

### 1. 哈希冲突是必然的

```
输入空间无限 → 输出空间有限 → 鸽巢原理 → 必然存在碰撞
```

安全性不在于「没有碰撞」，而在于**计算上无法找到碰撞**。

### 2. 已知被攻破的算法

| 算法 | 输出位 | 碰撞攻击复杂度 | 状态 |
|------|--------|----------------|------|
| **MD5** | 128 | 2^18（实际可秒破） | ❌ 已废弃 |
| **SHA-1** | 160 | 2^63（SHAttered 攻击） | ❌ 已废弃 |
| **SHA-256** | 256 | 2^128（理论） | ✅ 安全 |
| **SHA-3** | 可变 | 2^{n/2}（理论） | ✅ 安全 |
| **bcrypt** | 184 | — | ✅ 推荐用于密码 |

**MD5 为何被破解？**  
利用**差分密码分析**——攻击者构造特定输入差异，使其在压缩函数中传播并互相抵消，最终产生相同的哈希值。研究者找到了高效构造碰撞消息对的方法。

### 3. 长度扩展攻击（Length Extension Attack）

Merkle-Damgård 结构的固有缺陷。已知 `H(m)`，可以计算出 `H(m ‖ padding ‖ attacker_data)`，**无需知道 m 本身**。

```
SHA-256("secret" + "data") 泄露后
→ 攻击者可计算出 SHA-256("secret" + "data" + padding + "evil")
→ 在不知道 "secret" 的情况下伪造有效哈希
```

**防御**：使用 HMAC 或 SHA-3（海绵结构天然免疫）。

### 4. 可变对象作为字典键的反例

```python
# ❌ 经典陷阱：list 作为键
d = {}
d[[1, 2]] = "value"  # TypeError: unhashable type: 'list'

# ⚠️ 更隐蔽的陷阱：tuple 包含 list
d = {}
t = (1, [2, 3])
d[t] = "value"  # TypeError: unhashable type: 'list'
# tuple 本身不可变，但包含可变子元素，整体不可哈希
```

### 5. 不是什么时候都用哈希

- **密码存储**：不能用通用哈希（SHA-256），必须用 bcrypt/argon2/scrypt
- **哈希表**：Java HashMap 的扰动函数 `h ^ (h >>> 16)` 说明好的哈希函数 ≠ 好的哈希表索引——需要额外扰动增强低位随机性
- **数字签名**：签名的是 `hash(消息)`，而非消息本身（效率+安全性）
