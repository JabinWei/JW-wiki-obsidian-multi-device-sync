## 💡 核心概念

**元组的「不可变」是指元素引用不可变，而非被引用对象本身不可变。**

换句话说：元组存储的是一组**指针**，这些指针的指向不能改变，但指针指向的那个对象如果本身是可变的，你依然可以修改它。

---

## 🧠 心智模型：书架比喻

把元组想象成一个**钉在墙上的书架**：

| 书架（元组） | 格子 0 | 格子 1 | 格子 2 |
|---|---|---|---|
| 存放的东西 | 整数 `1` | 整数 `2` | 列表 `[3, 4]` |

- **引用不可变** = 每个格子里放的**是什么东西**不能换
- **引用对象可变** = 如果格子里放的是一本可以涂改的笔记本（列表），你可以在笔记本里写字

```python
t = (1, 2, [3, 4])

# ❌ 换掉格子里放的东西 → 改引用 → 报错
t[0] = 10          # TypeError
t[2] = [6, 7]      # TypeError

# ✅ 在笔记本里涂改 → 改引用对象 → 合法
t[2].append(5)      # t 变成 (1, 2, [3, 4, 5])
```

## 🔬 两条规则，一条红线

### 规则 1：可以修改元组中可变元素的内部状态

```python
# 列表
t = (1, [2, 3])
t[1].append(4)      # ✅ (1, [2, 3, 4])
t[1][0] = 99        # ✅ (1, [99, 3, 4])

# 字典
t = (1, {'a': 1})
t[1]['b'] = 2       # ✅ (1, {'a': 1, 'b': 2})

# 自定义对象
class Counter:
    def __init__(self): self.n = 0
    def inc(self): self.n += 1

t = (1, Counter())
t[1].inc()          # ✅ (1, <Counter n=1>)
```

### 规则 2：不能改变元素引用本身

```python
t = (1, 2, 3)
t[0] = 10           # ❌ TypeError
del t[0]            # ❌ TypeError
t.append(4)         # ❌ AttributeError
```

## ⚠️ 关键推论：可哈希性取决于元素

不是所有元组都能做字典键——包含不可哈希元素的元组自身也不可哈希：

```python
d = {}
d[(1, 2)] = 'ok'             # ✅
d[([1, 2], 3)] = 'fail'      # ❌ TypeError: unhashable type: 'list'

hash((1, 2))          # ✅ 正常返回
hash((1, [2, 3]))     # ❌ TypeError
```

## 🧪 经典陷阱

```python
t = (1, 2, [3, 4])
try:
    t[2] += [8]
except TypeError:
    pass
print(t)  # (1, 2, [3, 4, 8])  ← 列表被改了但代码仍抛异常！
```

`+=` 先原地 `extend`（成功），再尝试赋值回 `t[2]`（失败），导致"既改了又报错"。

---

## ❌ 常见误区

1. **「元组完全不可变」** → 不可变的是引用，不是引用目标
2. **「所有元组都可做字典键」** → 含可变元素的元组不可哈希
3. **「不可变 = 线程安全」** → 内部可变对象的修改不线程安全
4. **「元组和字符串一样不可变」** → 字符串是彻底的不可变；元组只锁第一层引用
