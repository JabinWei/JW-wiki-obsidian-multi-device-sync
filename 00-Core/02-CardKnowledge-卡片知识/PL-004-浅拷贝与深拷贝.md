# 浅拷贝与深拷贝

## 💡 核心概念

**浅拷贝（Shallow Copy）** 和 **深拷贝（Deep Copy）** 是对象复制的两种基本策略，核心区别在于对**嵌套引用**的处理：

- **浅拷贝**：只复制对象本身，内部嵌套的可变对象**共享引用**（新旧对象指向同一块内存）
- **深拷贝**：递归复制整个对象图，包括所有嵌套层级，**完全独立**

> 🔑 直接赋值不是拷贝——只是给原对象起了个别名（引用）。

```
原对象 A ──→ { name: "Jason", skills: ──→ ["Python", "Java"] }

浅拷贝 B ──→ { name: "Jason", skills: ──→ 同一个 ["Python", "Java"] }
                                              ↑
深拷贝 C ──→ { name: "Jason", skills: ──→ ["Python", "Java"] }  ← 独立副本
```

---

## 🔍 详细说明

### 一、为什么要区分浅/深拷贝？—— 从内存说起

在大多数编程语言中，对象字段分为两类：

| 字段类型 | 存储内容 | 复制行为 |
|----------|----------|----------|
| **值类型**（int, float, bool, string 等不可变对象） | 数据本身 | 复制即独立 |
| **引用类型**（list, dict, object, array 等可变对象） | 指向堆内存的**指针** | 复制的是指针，指向同一块内存 |

浅拷贝只复制第一层的值类型，遇到引用类型只复制指针。深拷贝则递归追踪所有引用，在堆上创建全新副本。

### 二、各语言实现对比

| 语言 | 浅拷贝 | 深拷贝 |
|------|--------|--------|
| **Python** | `copy.copy()` / `list[:]` / `dict.copy()` | `copy.deepcopy()` |
| **Java** | `Object.clone()`（默认） / 手动逐字段复制 | 手动递归 clone / 序列化反序列化 / 第三方库 |
| **JavaScript** | `Object.assign()` / 展开运算符 `{...obj}` | `structuredClone()` / `JSON.parse(JSON.stringify(obj))` |
| **C++** | 默认拷贝构造函数（成员逐一复制，指针共享） | 自定义拷贝构造函数（为指针成员 `new` 新内存） |

### 三、通用判断原则

1. **对象只包含不可变/值类型** → 浅拷贝就够，浅拷贝等价于深拷贝
2. **包含一层可变容器**（如列表的列表）→ 浅拷贝只复制外层，内层仍共享
3. **多层嵌套或循环引用** → 必须深拷贝，且需要能处理循环引用的实现（如 Python 的 `deepcopy` 内部维护 `memo` 字典防死循环）

## 📎 例子

### Python

```python
import copy

original = {'name': 'Jason', 'skills': ['Python', 'Java']}

# 浅拷贝：skills 列表共享
shallow = copy.copy(original)
shallow['skills'].append('C++')
print(original['skills'])  # ['Python', 'Java', 'C++'] ← 被影响了！

# 深拷贝：完全独立
deep = copy.deepcopy(original)
deep['skills'].append('Go')
print(original['skills'])  # ['Python', 'Java', 'C++'] ← 不受影响
```

### JavaScript

```javascript
const original = { name: 'Jason', skills: ['Python', 'Java'] };

// 浅拷贝：skills 数组共享
const shallow = { ...original };
shallow.skills.push('C++');
console.log(original.skills);  // ['Python', 'Java', 'C++'] ← 被影响！

// 深拷贝
const deep = structuredClone(original);
deep.skills.push('Go');
console.log(original.skills);  // ['Python', 'Java', 'C++'] ← 不受影响
```

### Java

```java
// 浅拷贝：clone() 默认行为
class Person implements Cloneable {
    String name;
    List<String> skills;  // 引用类型，浅拷贝只复制引用

    @Override
    protected Person clone() {
        return (Person) super.clone();  // 浅拷贝
    }
}

// 深拷贝：手动复制可变字段
@Override
protected Person clone() {
    Person p = (Person) super.clone();
    p.skills = new ArrayList<>(this.skills);  // 复制内部列表
    return p;
}
```

### C++

```cpp
#include <iostream>
#include <cstring>
using namespace std;

class Person {
public:
    char* name;
    int age;

    // 普通构造函数
    Person(const char* n, int a) {
        name = new char[strlen(n) + 1];
        strcpy(name, n);
        age = a;
    }

    // 默认拷贝构造函数（浅拷贝）—— 编译器自动生成
    // Person(const Person& other)
    //     : name(other.name), age(other.age) { }
    // 问题：name 指针被两个对象共享！

    // 自定义拷贝构造函数（深拷贝）—— 必须手动写
    Person(const Person& other) {
        name = new char[strlen(other.name) + 1];  // 分配新内存
        strcpy(name, other.name);                  // 复制内容
        age = other.age;
    }

    // 深拷贝的赋值运算符也必不可少
    Person& operator=(const Person& other) {
        if (this == &other) return *this;          // 防止自赋值
        delete[] name;                             // 释放旧内存
        name = new char[strlen(other.name) + 1];   // 分配新内存
        strcpy(name, other.name);                  // 复制内容
        age = other.age;
        return *this;
    }

    ~Person() { delete[] name; }                   // 析构函数释放资源
};

// 验证
int main() {
    Person p1("Jason", 25);
    Person p2 = p1;            // 调用深拷贝构造函数

    p2.name[0] = 'M';          // 修改 p2 的名字
    cout << p1.name << endl;   // "Jason" — 不受影响
    cout << p2.name << endl;   // "Mason"
}
```

> ⚠️ **C++ 三法则**：如果类需要自定义析构函数、拷贝构造函数或拷贝赋值运算符三者之一，则几乎肯定三者都需要。上例中 `Person` 管理动态内存（`name` 指针），因此三个全部重写。

## ⚡ 反例 / 边界

### 1. 并非所有语言都内置深拷贝

- **Java** 没有语言级别的 `deepcopy`，需要手动递归实现或借助序列化
- **JavaScript** 的 `structuredClone` 是近年才标准化，老旧做法 `JSON.parse(JSON.stringify(obj))` 会丢失函数、`Date`、`undefined`、循环引用

### 2. 浅拷贝 ≠ 赋值

```python
# 赋值：同一个对象
a = [1, 2, [3, 4]]
b = a
b is a  # True ← 只是别名

# 浅拷贝：新对象，但内部引用共享
c = list(a)  # 或 a.copy() 或 a[:]
c is a       # False ← 不同容器
c[2] is a[2]  # True  ← 但嵌套列表是同一个！
```

### 3. 循环引用

深拷贝必须处理对象之间的循环引用，否则会无限递归。

```python
import copy

a = []
b = [a]
a.append(b)  # a → b → a → b → ... 循环引用

c = copy.deepcopy(a)  # ✅ Python 内部用 memo 字典防死循环
```

### 4. 不可变对象：浅拷贝可以「省略」内部复制

Python 的 `copy.copy()` 对 `tuple` 直接返回自身（因为不可变，无需复制）。但 `copy.deepcopy()` 仍然会复制内部的元素对象。

```python
import copy

t = (1, [2, 3])
shallow = copy.copy(t)    # 返回 t 自身
print(shallow is t)       # True

deep = copy.deepcopy(t)   # 创建新 tuple，内部 list 也复制
print(deep is t)           # False
print(deep[1] is t[1])     # False
```
