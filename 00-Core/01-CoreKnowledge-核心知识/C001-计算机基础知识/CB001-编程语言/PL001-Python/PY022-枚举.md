# PY022-枚举

## 🎯 学习目标

1. 什么是枚举？为什么使用枚举而不是普通常量？
2. `Enum`、`IntEnum`、`StrEnum`、`Flag`、`IntFlag` 之间有什么区别，各自的使用场景是什么？
3. 如何通过 `auto()` 自动赋值？`auto()` 的赋值规则有哪些？
4. 枚举成员有哪些核心属性（`name`、`value`）和方法（`__members__`、`__contains__`）？
5. 如何遍历、比较、查找枚举成员？枚举成员与普通值的互操作有哪些注意点？

## 📖 前置知识

- [[PY014-函数]] — 理解函数定义与调用，枚举的方法本质上是类方法。

## 📚 核心内容

枚举是 Python 中表示**固定有限集合**的标准方式。它用有意义的名称代替魔术数字（magic number），提升代码可读性和类型安全。

---

### 基础枚举：`Enum`

#### 1. 创建枚举

```python
from enum import Enum

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3
```

#### 2. 访问枚举成员

```python
# 通过名称访问
print(Color.RED)        # Color.RED

# 通过值访问
print(Color(1))         # Color.RED

# 通过字典式访问
print(Color['RED'])     # Color.RED
```

#### 3. 核心属性：`name` 和 `value`

```python
member = Color.RED
print(member.name)      # 'RED'
print(member.value)     # 1
```

#### 4. 遍历与成员检查

```python
# 遍历所有成员
for color in Color:
    print(color.name, color.value)

# 检查值或名称是否属于枚举
print('RED' in Color.__members__)  # True
print(1 in Color._value2member_map_)  # True (内部使用)
```

> **`__members__` 与 `_value2member_map_` 的区别：**
>
> 两者都是枚举类的内部映射字典，方向不同：
>
> | 属性 | 映射方向 | 示例 |
> |---|---|---|
> | `__members__` | **名称 → 成员** | `Color.__members__['RED']` → `Color.RED` |
> | `_value2member_map_` | **值 → 成员** | `Color._value2member_map_[1]` → `Color.RED` |
>
> `_value2member_map_` 名字以 `_` 开头，说明它是**内部实现细节**，日常开发中更推荐用异常捕获：`try: Color(1) except ValueError: ...`，不直接依赖私有属性。

#### 5. 枚举的独特规则

- **不可实例化**：不能 `Color()` 创建新成员。
- **值不可变**：成员的值是只读的，不能修改。
- **唯一性检查**：默认允许不同名称映射到相同值；如需防止重复值，使用 `@unique` 装饰器。
- **不可比较大小**：`Color.RED < Color.BLUE` 会报错，但 `is` 和 `==` 可用于身份和等值比较。

### 枚举变体与使用场景对比

| 变体            | 基类             | 核心特征                                  | 典型场景                |     |
| ------------- | -------------- | ------------------------------------- | ------------------- | --- |
| **`Enum`**    | —              | 标准枚举，成员是独立的枚举实例                       | 状态机、选项列表等绝大多数场景     |     |
| **`IntEnum`** | `int` + `Enum` | 成员同时也是 `int`，可直接参与整数运算和比较             | 与 C 库、数据库整数码交互      |     |
| **`StrEnum`** | `str` + `Enum` | 成员同时也是 `str`，可直接用于字符串替换（Python 3.11+） | 配置文件、API 参数、HTTP 方法 |     |
| **`Flag`**    | `Enum`         | 支持位运算组合（\|、`&`、`^`、`~`），成员值应为 2 的幂    | 多选项组合（如文件权限、窗口样式）   |     |
| **`IntFlag`** | `int` + `Flag` | Flag 的 int 版本，支持位运算 + 整数运算            | 系统调用、C 扩展中的位掩码组合    |     |

**各变体使用示例：**

```python
from enum import Enum, IntEnum, StrEnum, Flag, IntFlag, auto

# --- Enum：标准枚举 ---
class Status(Enum):
    PENDING = 'pending'
    RUNNING = 'running'
    DONE = 'done'

print(Status.PENDING.value)  # 'pending'

# --- IntEnum：与 int 互操作 ---
class Priority(IntEnum):
    LOW = 1
    MEDIUM = 2
    HIGH = 3

print(Priority.LOW + 1)       # 2 (可以直接运算)
print(Priority.LOW < Priority.HIGH)  # True (可以直接比较)

# --- StrEnum：与 str 互操作 (Python 3.11+) ---
class Method(StrEnum):
    GET = 'GET'
    POST = 'POST'

print(Method.GET + ' /api')   # 'GET /api'
print(f'{Method.POST}')       # 'POST' (可直接用于 f-string)

# --- Flag：位掩码组合 ---
class Permission(Flag):
    READ = 1      # 二进制: 001
    WRITE = 2     # 二进制: 010
    EXECUTE = 4   # 二进制: 100

rw = Permission.READ | Permission.WRITE
print(rw)                     # Permission.READ|WRITE
print(Permission.READ in rw)  # True
print(Permission.EXECUTE in rw)  # False

# --- IntFlag：位掩码 + int 运算 ---
class Style(IntFlag):
    BOLD = 1
    ITALIC = 2
    UNDERLINE = 4

print(Style.BOLD | Style.ITALIC)  # 3
```

### 自动赋值：`auto()`

当不需要手动指定值时，使用 `auto()` 让 Python 自动生成值。

```python
from enum import Enum, auto

class Color(Enum):
    RED = auto()    # 1
    GREEN = auto()  # 2
    BLUE = auto()   # 3
```

**`auto()` 的赋值规则：**

| 使用场景 | 生成规则 |
|----------|----------|
| 普通 `Enum` | 递增整数：1, 2, 3, … |
| 混合手动值 + `auto()` | 在最大手动值后继续递增 |
| 在 `Flag` / `IntFlag` 中 | 自动生成 2 的幂：1, 2, 4, 8, … |
| 自定义 `_generate_next_value_()` | 由你定义的方法决定 |

**自定义 `auto()` 生成规则：**

如果不满足于默认的递增整数或 2 的幂，可以重写 `_generate_next_value_()` 方法来定义自己的生成逻辑。

`_generate_next_value_()` 是 `auto()` 背后的钩子方法。每次 `auto()` 被调用时，Python 内部就会调用这个方法，把相关信息传进来，用它的返回值作为该成员的枚举值。

**参数详解：**

| 参数 | 含义 | 示例（假设定义 `class Animal(MyEnum):`） |
|------|------|------------------------------------------|
| `name` | 当前成员的**名称**（字符串） | `'CAT'`、`'DOG'`、`'BIRD'` |
| `start` | 枚举的**起始值**，`class` 语法下固定为 1，只有函数式 `Enum('Name', 'A B C', start=10)` 才能改为其他值 | `1` |
| `count` | 当前是**第几个**成员（从 1 开始） | CAT 是第 1 个，DOG 是第 2 个 |
| `last_values` | 到目前为止**已经赋过的值**列表 | 处理 DOG 时，`last_values = ['cat']` |

> **新手理解关键**：不需要手写 `count` 和 `last_values`。你只需定义规则（比如"用小写名称作为值"），每次 `auto()` 被调用时 Python 会自动传入当前状态。
>
> **为什么 `start` 永远是 1？**
>
> `start` 的存在是为了函数式 API 准备的：`Enum('MyEnum', 'A B C', start=100)` 可以指定起始值。但 `class` 语法没有提供传 `start` 的地方，所以 Python 固定传 `1`。
>
> 因此，在 `class` 语法下定义自定义生成规则时，`start` 参数基本用不上。真正有用的是 `name`（成员名）和 `count`（第几个成员），它们足以覆盖绝大多数自定义场景。

**示例：用小写名称作为枚举值**

```python
from enum import Enum, auto

class MyEnum(Enum):
    def _generate_next_value_(name, start, count, last_values):
        """每当 auto() 被调用时，返回 name 的小写形式作为值"""
        return name.lower()

class Animal(MyEnum):
    CAT = auto()    # Python 调用 _generate_next_value_('CAT', 1, 1, []) → 返回 'cat'
    DOG = auto()    # Python 调用 _generate_next_value_('DOG', 1, 2, ['cat']) → 返回 'dog'
    BIRD = auto()   # Python 调用 _generate_next_value_('BIRD', 1, 3, ['cat', 'dog']) → 返回 'bird'

print(Animal.CAT.value)   # 'cat'
print(Animal.DOG.value)   # 'dog'
print(Animal.BIRD.value)  # 'bird'
```

**用"看懂执行过程"的视角重新理解上面的代码：**

1. Python 执行到 `CAT = auto()` → 调用 `_generate_next_value_('CAT', 1, 1, [])` → 返回 `'cat'` → CAT 的值就是 `'cat'`
2. Python 执行到 `DOG = auto()` → 调用 `_generate_next_value_('DOG', 1, 2, ['cat'])` → 返回 `'dog'` → DOG 的值就是 `'dog'`
3. Python 执行到 `BIRD = auto()` → 同样流程 → 返回 `'bird'`

**另一个实用示例：用自增 ID 字符串作为值**

```python
class OrderStatus(Enum):
    def _generate_next_value_(name, start, count, last_values):
        return f'{count:03d}_{name}'  # 如 '001_PENDING'

    PENDING = auto()     # '001_PENDING'
    SHIPPED = auto()     # '002_SHIPPED'
    DELIVERED = auto()   # '003_DELIVERED'

print(OrderStatus.PENDING.value)  # '001_PENDING'
```

### 枚举的高级功能

#### 1. 函数式 API：便捷创建简单枚举

```python
# 函数式创建（适合简单场景）
Animal = Enum('Animal', 'CAT DOG BIRD')
print(list(Animal))  # [<Animal.CAT: 1>, <Animal.DOG: 2>, <Animal.BIRD: 3>]

# 带值创建
Color = Enum('Color', [('RED', '#FF0000'), ('GREEN', '#00FF00')])
print(Color.RED.value)  # '#FF0000'
```

#### 2. `@unique` 装饰器：防止重复值

```python
from enum import unique

@unique
class Status(Enum):
    ACTIVE = 1
    INACTIVE = 2
    # ARCHIVED = 2  # ❌ ValueError: duplicate values found
```

#### 3. 给枚举成员添加方法

```python
class Operation(Enum):
    ADD = '+'
    SUB = '-'

    def apply(self, a, b):
        if self == Operation.ADD:
            return a + b
        elif self == Operation.SUB:
            return a - b

print(Operation.ADD.apply(3, 2))  # 5
```

### 枚举 vs 普通常量 vs 字典

| 维度 | 枚举 `Enum` | 普通常量 (`RED = 1`) | 字典 `{'RED': 1}` |
|------|-------------|---------------------|--------------------|
| 类型安全 | ✅ 强类型 | ❌ 魔术数字 | ❌ 字符串键 |
| 遍历成员 | ✅ `for m in Enum` | ❌ 需手动收集 | ✅ `dict.items()` |
| 反向查找 | ✅ `Enum(value)` | ❌ | ❌ 需 `dict` 反转 |
| 不可变性 | ✅ 自动保护 | ❌ 可随意修改 | ❌ 可随意修改 |
| 自文档化 | ✅ `repr()` 输出可读 | ❌ | ⚠️ 仅键值对 |
| 位运算组合 | ✅ `Flag` | ❌ | ❌ |

使用 `IntEnum` 创建 HTTP 状态码枚举，包含 OK(200)、NOT_FOUND(404)、SERVER_ERROR(500)，并遍历打印。

    SERVER_ERROR = 500

OK: 200

## 🤔 常见误区

### 误区 1：认为枚举值可以随意修改

**错误认知**：`Color.RED.value = 100` 可以修改已有的枚举值。

**事实**：枚举成员是单例对象，其 `value` 属性是只读的。尝试修改会抛出 `AttributeError`。枚举一旦定义，成员和值的映射关系就固定了。

### 误区 2：用 `auto()` 但期望在 `Flag` 中也是递增整数

**错误认知**：在 `Flag` 中使用 `auto()` 也会生成 1, 2, 3, …。

**事实**：`Flag` 和 `IntFlag` 中的 `auto()` 自动生成 2 的幂（1, 2, 4, 8, …），因为位掩码需要每个标志位独立。如果需要递增整数，用普通 `Enum` 而非 `Flag`。

### 误区 3：枚举值可以进行比较排序

**错误认知**：`Color.RED < Color.GREEN` 可以判断顺序。

**事实**：`Enum` 不支持排序比较（`<`、`>`、`<=`、`>=`），会抛出 `TypeError`。这是为了避免将枚举当作有序集合使用。`IntEnum` 和 `IntFlag` 是例外，因为它们继承自 `int`。

### 误区 4：把枚举当作字典使用

**错误认知**：`Color['RED']` 返回的是值 `1`。

**事实**：`Color['RED']` 返回的是枚举成员 `Color.RED`。要获取值需要使用 `.value` 属性：`Color['RED'].value`。
