## 📚 核心内容

在编程的世界里，**函数（Function）** 是构建复杂程序的基石。它不仅仅是一段带名字的代码块，更是我们实现代码复用、逻辑解耦的利器。

### 定义与文档字符串

定义一个函数非常简单，使用 `def` 关键字即可。但一个好的函数，必须包含清晰的**文档字符串（Docstring）**。

- ** `def` 关键字**：告诉 Python 我要定义一个函数了。
- **文档字符串**：位于函数头下方的三引号（或双引号）内容。它不是普通的注释，而是用于描述函数“做什么”的说明书，这对于后期维护至关重要。它会被存储在函数的 `__doc__` 属性中，可以通过 `help()` 函数调用，是自动化文档工具（如 Sphinx）的基础。

```python
def greet_user():
    """显示简单的问候语"""
    # 这里的三引号内容就是文档字符串，可以通过 help(greet_user) 查看
    print("Hello!")

greet_user() # 调用函数
```

> **小贴士**：养成写文档字符串的习惯，这会让你的代码看起来更专业，也更容易被他人理解。

### 形参与实参

函数不是封闭的孤岛，它需要接收外部数据进行处理。这里涉及两个概念：

- **形参 (Parameter)**：函数定义时的变量，是接口声明。
- **实参 (Argument)**：调用函数时传递的具体信息（常量或变量）。

Python 提供了灵活的传参方式，让你不再为参数顺序头疼：

| 传参方式       | 描述                      | 注意事项                             |
| -------------- | ------------------------- | ------------------------------------ |
| **位置实参**   | 实参与形参位置一一对应    | 顺序必须严格一致，否则逻辑会出错。   |
| **关键字实参** | 通过 `形参名=实参值` 传递 | 无需关注顺序，但形参名必须拼写正确。 |
| **默认值**     | 定义时给形参赋默认值      | 若调用时未传参，则使用默认值。       |

**Python 的参数传递机制**
Python 的参数传递既不是单纯的“值传递”也不是“引用传递”，而是**“对象引用传递”**。

- 如果传递的是**不可变对象**（如数字、字符串、元组），函数内部修改形参不会影响外部实参（类似值传递）。
- 如果传递的是**可变对象**（如列表、字典），函数内部修改形参会直接影响外部实参（类似引用传递）。

**️关于默认值的避坑指南：**
如果你给某个形参设置了默认值，那么**该形参后面的所有形参都必须设置默认值**。这是 Python 的语法规则，否则会报错。

> **可变默认参数**
> 千万不要使用可变对象（如列表 `[]` 或字典 `{}`）作为默认参数！
> 因为默认参数在函数定义时只计算一次。如果修改了它，后续调用都会受到影响。
> **错误写法**：`def add_item(item, lst=[]): ...`
> **正确写法**：`def add_item(item, lst=None): if lst is None: lst = []`

### 传递任意数量的实参

在实际开发中，我们往往无法预知用户会传递多少个参数。Python 提供了两种强大的机制来处理这种情况：

#### 1. 接收任意数量的位置实参 (`*args`)

在形参前加星号 `*`，Python 会将所有接收到的实参打包成一个**元组 (Tuple)**。

```python
def make_pizza(*toppings):
    """打印顾客点的所有配料"""
    print(toppings) # toppings 在这里是一个元组

make_pizza('pepperoni')
# 输出: ('pepperoni',)
```

#### 2. 接收任意数量的关键字实参 (`**kwargs`)

在形参前加双星号 `**`，Python 会将接收到的键值对打包成一个**字典 (Dictionary)**。

```python
def build_profile(first, last, **user_info):
    """创建一个字典，其中包含我们知道的有关用户的一切"""
    user_info['first_name'] = first
    user_info['last_name'] = last
    return user_info
```

#### 3. 混合使用与位置原则

如果同时使用普通参数和 `*args`，建议将 `*args` 放在最后。如果 `*args` 后面还有参数，那么后续参数必须使用**关键字实参**的方式进行传递。

**参数定义的顺序**
在定义复杂函数时，参数必须遵循以下顺序：
`def func(标准参数, *args, 默认参数, **kwargs):`

### 函数返回值

函数处理完数据后，通常需要将结果返回给调用者。

**有返回值的情况：**

```python
def get_formatted_name(first_name, last_name):
    full_name = f"{first_name} {last_name}"
    return full_name.title() # return 语句结束函数执行并返回值
```

**没有返回值的情况 (注意事项)：**
如果函数体内没有 `return` 语句，函数执行完毕后默认返回 `None`。这类函数通常用于执行动作（如打印信息），而不是计算数据。

**返回多个值**
Python 允许函数返回多个值，本质上是返回了一个**元组**。
`return a, b` 等同于 `return (a, b)`。调用时可以使用 `x, y = func()` 进行解包。

当只需要部分返回值时，可以用 `_` 丢弃不需要的值：

```python
def get_user():
    return "Alice", "shh", 25

name, _, age = get_user()  # 丢弃中间值
_, result = some_func()    # 只要第二个返回值
```

> 详见 [[PL-PY-003-Python中下划线占位符的使用]]

**注意点：**

如果函数参数中有一个默认参数 `listv=[]` ，这个参数在后续函数多次调用的时候并不会重复创建，只在首次调用的时候创建一次。

```python
def add_list(value, listv=[]):
    listv.append(value)
    return listv

glist = []
v1 = add_list(1, glist)

v2 = add_list(2)
v3 = add_list(3)

print(v1, id(v1))
print(v2, id(v2))
print(v3, id(v3))

# 输出：
# [1] 4423672064
# [2, 3] 4423671936
# [2, 3] 4423671936
```

### 函数作用域

在 Python 编程中，很多初学者（甚至是有经验的开发者）都会遇到一个经典问题：“为什么我的变量在这里找不到？”或者“为什么修改了这个列表，另一个地方的数据也跟着变了？”。这些问题的根源，往往指向了 Python 中两个至关重要但容易被混淆的概念：**命名空间（Namespace）**与**作用域（Scope）**。

#### 命名空间：变量的“容器”

简单来说，命名空间就是一个从“名称”到“对象”的映射表（你可以把它理解为一个字典）。它的核心作用是**避免命名冲突**——在不同的命名空间里，可以有同名的变量，彼此互不干扰。

Python 中的命名空间主要分为以下四类，它们的生命周期各不相同：

| 命名空间 | 创建时机 | 销毁时机 | 包含内容 |
|---|---|---|---|
| **Built-in** | 解释器启动时，最先创建 | 解释器进程终止 | `print`、`len`、`Exception` 等内置函数和异常 |
| **Global** | 模块被 `import` 时创建，缓存在 `sys.modules` 中 | 模块对象引用计数归零时（通常伴随进程退出，但可手动清理 `sys.modules` 释放） | 模块层级的变量、函数、类 |
| **Enclosing** | 外层函数被调用时，为其创建的局部命名空间 | 外层函数执行完毕返回时 | 外层函数内的变量，对内层嵌套函数可见 |
| **Local** | 当前函数被调用时创建 | 当前函数执行完毕（返回或抛异常）时 | 当前函数内的变量和形参 |

> **按生命周期从长到短排列**：Built-in → Global → Enclosing → Local

> **补充：Global 命名空间的实际回收问题**
> Global 命名空间虽然绑定了模块对象，但由于 `sys.modules` 缓存的存在，模块一经导入便持续持有引用——所以实际开发中，"不再使用的模块"并不会自动释放。真正要释放需要手动 `del sys.modules['module_name']` 切断缓存引用，GC 才会在引用计数归零后回收。

#### 作用域与 LEGB 规则

如果说命名空间是“容器”，那么作用域就是“访问规则”。它决定了你在代码的某个位置，能够直接访问到哪个命名空间里的变量。

当你在代码中引用一个变量时，Python 会按照 **L → E → G → B** 的顺序**由内向外**逐层查找变量，**找到即停**：

1. **L (Local)**：当前函数的局部命名空间。最近的定义，优先级最高。
2. **E (Enclosing)**：逐层向外查找**每一个**外层函数的局部命名空间。例如三层嵌套时，从内到外依次搜 `inner → middle → outer`，任意一层找到就停止。
3. **G (Global)**：当前模块的全局命名空间。注意：**每个 `.py` 文件有自己独立的全局命名空间**，模块 A 的全局变量不会出现在模块 B 的 LEGB 查找路径中。
4. **B (Built-in)**：Python 内置命名空间。这是最后兜底的一层，也是范围最大的一层。

四层都找不到 → `NameError: name 'xxx' is not defined`

> **为什么是这个顺序？** "最近的优先"这个设计确保了两件事：(1) 函数内的局部变量不会被外层同名变量意外覆盖——数据安全；(2) 你可以在局部放心使用 `len`、`max` 等名字，不影响内置函数——灵活性。

**💡 举个例子：**

```python
x = "全局变量 (G)"  # 全局作用域

def outer():
    x = "嵌套变量 (E)"  # 嵌套作用域（outer 的局部命名空间）
    def inner():
        x = "局部变量 (L)"  # 局部作用域（inner 的局部命名空间）
        print(x)  # LEGB 查找: L ✓ (命中，停止)
    inner()
    # 此时 inner 已返回，inner 的 L 命名空间已销毁
    print(x)      # LEGB 查找: L ✗ (inner 已销毁) → E ✓ (命中，停止)

outer()
# 此时 outer 已返回，outer 的 E 命名空间也已销毁
print(x)          # LEGB 查找: L ✗ → E ✗ → G ✓ (命中，停止)
```

*输出结果依次为：局部变量 (L) → 嵌套变量 (E) → 全局变量 (G)*

#### 跨域修改变量：global 与 nonlocal

在函数内部，我们可以轻松**读取**外层（E 层或 G 层）的变量，但如果想要**修改**它们，就需要特别注意了。

- **赋值即定义**：如果在函数内部直接对某个变量赋值（例如 `x = 1`），Python 会默认将其视为该函数的**局部变量**。如果此时你想引用外层的同名变量，就会报错 `UnboundLocalError`。
- **global 关键字**：用于在函数内部声明并修改**全局作用域（G）**的变量。
- **nonlocal 关键字**：用于在嵌套函数中声明并修改**最近的外层函数作用域（E）**的变量。

```python
count = 0  # 全局变量

def outer():
    total = 10  # 外层变量
    def inner():
        global count
        nonlocal total
        count += 1
        total += 5
        print(f"内层 - count: {count}, total: {total}")
    inner()
    print(f"外层 - total: {total}")

outer()
print(f"全局 - count: {count}")
```

#### 避坑指南：默认参数的陷阱

在理解作用域时，有一个极易踩坑的知识点：**函数的默认参数是在函数定义时求值的，且只计算一次**。

如果你使用了可变对象（如列表、字典）作为默认参数，很容易引发 Bug：

```python
# ❌ 错误示范
def add_item(item, bag=[]):
    bag.append(item)
    return bag

print(add_item(1))  # 输出: [1]
print(add_item(2))  # 预期输出 [2]，实际却输出: [1, 2]！
```

这是因为 `bag` 这个列表在函数定义时就被创建了，后续所有的调用都在共用这同一个列表对象。

**✅ 正确姿势**：使用 `None` 作为占位符，在函数内部初始化可变对象。

```python
def add_item(item, bag=None):
    if bag is None:
        bag = []
    bag.append(item)
    return bag
```

### Lambda 表达式

在 Python 中，除了使用 `def` 定义命名函数外，还有一种特殊的函数叫做**匿名函数**，即 Lambda 表达式。

#### 什么是 Lambda？

Lambda 是一个**没有名字**、**只能写一行代码**的微型函数。它适合用于那些简单、一次性、不需要复用的逻辑场景。

**语法格式：**

```python
lambda 参数: 表达式
```

#### Lambda 与普通函数的对比：

| 特性         | 普通函数 (def)           | Lambda 表达式                       |
| ------------ | ------------------------ | ----------------------------------- |
| **名称**     | 有名字                   | 匿名（无名字）                      |
| **代码体**   | 可以写多行，包含各种逻辑 | **只能写一行**表达式                |
| **返回值**   | 需要显式写 `return`      | **自动返回**表达式结果，无需 return |
| **适用场景** | 复杂业务逻辑、需要复用   | 简单的数据处理、作为参数传递        |

**代码示例：**

```python
# 普通函数写法
def add(x, y):
    return x + y

# Lambda 写法
add_lambda = lambda x, y: x + y

print(add(2, 3))       # 输出: 5
print(add_lambda(2, 3)) # 输出: 5
```

#### Lambda 的参数定义规则

Lambda 表达式虽然简单，但其参数定义规则与普通函数（`def`）高度一致，只是省略了括号。

1. **无参数**：
   如果逻辑不需要输入，可以定义空参数。

   ```python
   # 定义一个返回固定值的 lambda
   get_pi = lambda: 3.14159
   print(get_pi()) # 输出: 3.14159
   ```

2. **多个位置参数**：
   参数之间用逗号 `,` 分隔。

   ```python
   # 计算长方形面积
   area = lambda length, width: length * width
   print(area(5, 3)) # 输出: 15
   ```

3. **默认参数**：
   和普通函数一样，Lambda 也支持为参数指定默认值。

   ```python
   # 定义一个带有默认乘数的 lambda
   multiplier = lambda x, factor=2: x * factor
   print(multiplier(5))      # 使用默认值，输出: 10
   print(multiplier(5, 3))   # 覆盖默认值，输出: 15
   ```

4. **可变参数 (`*args` 和 `**kwargs`)**：
   Lambda 也支持接收任意数量的参数，这在编写高阶函数时非常有用。

   ```python
   # 接收任意数量的位置参数并求和
   sum_all = lambda *args: sum(args)
   print(sum_all(1, 2, 3, 4)) # 输出: 10
   
   # 接收任意数量的关键字参数
   print_kwargs = lambda **kwargs: [f"{k}={v}" for k, v in kwargs.items()]
   print(print_kwargs(a=1, b=2)) # 输出: ['a=1', 'b=2']
   ```

> **️ 注意**：虽然 Lambda 写起来很酷，但不要滥用。如果逻辑稍微复杂一点，请务必使用 `def` 定义普通函数，以保证代码的可读性。

### 逻辑解耦

掌握了函数的语法只是第一步，如何用好它们，写出高质量的代码，则需要理解**逻辑解耦**的思想。

#### 什么是解耦？

简单来说，解耦就是**降低代码不同部分之间的依赖关系**。一个高度解耦的系统，其各个模块就像独立的积木块，可以轻松地被替换、修改或复用，而不会轻易“牵一发而动全身”。

- **紧耦合**：就像一列焊死的火车，一个车厢出问题，整列车都动不了。
- **松耦合**：就像由独立服务组成的生态系统，通过定义好的接口通信。

#### 如何实现逻辑解耦？

关键在于**依赖倒置原则**：**高层模块不应该依赖于低层模块，二者都应该依赖于抽象**。

**反面教材：紧耦合的代码**

```python
class ReportGenerator:
    def generate(self, data):
        # 问题：直接依赖了具体的 ExcelExporter
        # 如果以后想换成 PDF 导出，必须修改这里的代码，违反了“开闭原则”
        exporter = ExcelExporter()
        exporter.export(data)
```

**正面教材：松耦合的代码（依赖注入）**

```python
class ReportGenerator:
    def __init__(self, exporter): # 注入抽象接口
        self._exporter = exporter

    def generate(self, data):
        # 只依赖 exporter 接口，不关心具体是 Excel 还是 PDF
        self._exporter.export(data)
```

这种将依赖从外部注入的设计模式，被称为**依赖注入（Dependency Injection）**，是实现逻辑解耦最常用和有效的方法之一。

### 模块 (Modules)

当函数越来越多时，将所有代码堆在一个文件里会变得难以维护。这时我们需要引入**模块**的概念。

**导入模块的四种姿势：**

1. **导入整个模块**：`import module_name`
   - 调用时需加前缀：`module_name.func()`
2. **导入特定函数**：`from module_name import function_name`
   - 调用时直接：`function_name()`
3. **使用别名**：`import module_name as md`
   - 简化长模块名或避免冲突。
4. **导入所有**：`from module_name import *` (慎用)
   - **风险**：会将模块中所有公开名称导入当前命名空间，极易导致**命名覆盖**（例如不小心覆盖了内置函数 `open` 或 `list`），且难以追踪变量来源。

> 📦 **深入学习**：本节仅对模块做了简要介绍。关于 Python 模块的完整知识（导入方式对比、标准库模块详解、包管理、`__name__` 机制等），详见 [[PY019-模块]]。

### 练习 2：可变默认参数的陷阱

以下代码两次调用 `append_to_list(1)` 和 `append_to_list(2)`，分别输出什么？为什么会这样？请写出修复后的代码。

```python
def append_to_list(item, target=[]):
    target.append(item)
    return target
```

**答案：**
```python
# 原始代码的输出：
print(append_to_list(1))  # 输出: [1]
print(append_to_list(2))  # 输出: [1, 2] —— 共享了同一个默认列表！

# 修复后的正确写法：
def append_to_list(item, target=None):
    if target is None:
        target = []
    target.append(item)
    return target

print(append_to_list(1))  # 输出: [1]
print(append_to_list(2))  # 输出: [2]
```

**解释**：默认参数在函数定义时只计算一次，可变对象 `[]` 在多次调用间被共享。使用 `None` 作为哨兵值，在函数体内重新初始化，避免状态污染。

### 练习 3：使用 *args 和 **kwargs

编写函数 `describe_person`，接收必选参数 `name` 和 `age`，以及任意额外的信息（通过 `**kwargs`），返回格式化的描述字符串。再编写函数 `sum_numbers`，接收任意数量的数字，返回它们的和。

**答案：**
```python
def describe_person(name, age, **extra):
    result = f"{name}，今年 {age} 岁"

### 练习 4：LEGB 作用域查找

以下代码的输出是什么？请按照 LEGB 规则逐层解释。

name = "Global"

def outer():
    name = "Enclosing"
    def inner():
        print(name)
    inner()

outer()
```

**答案：**
输出为 `Enclosing`。

LEGB 查找顺序：
1. **L (Local)**：`inner()` 函数内部没有定义 `name`，跳过。
2. **E (Enclosing)**：外层函数 `outer()` 中定义了 `name = "Enclosing"`，找到，使用此值。
3. ~~G (Global)~~：不再继续查找。
4. ~~B (Built-in)~~：不再继续查找。

### 练习 5：global 与 nonlocal 的实际应用

阅读以下代码，补全输出结果。然后解释 `global` 和 `nonlocal` 各自影响了哪个变量。

```python
x = 10

def outer():
    x = 20
    y = 30
    def inner():
        global x
        nonlocal y
        x += 1
        y += 1
        print(f"inner: x={x}, y={y}")
    inner()
    print(f"outer: x={x}, y={y}")

outer()
print(f"global: x={x}")
```

**答案：**
```
inner: x=11, y=31
outer: x=20, y=31
global: x=11
```

**解释**：
- `global x`：`inner` 中的 `x` 指向全局 `x`（原本 = 10），执行 `x += 1` 后全局 `x` 变为 11。`outer` 中的 `x = 20` 没有被修改。
- `nonlocal y`：`inner` 中的 `y` 指向 `outer` 中的 `y`（原本 = 30），执行 `y += 1` 后 `outer` 中的 `y` 变为 31。

### 练习 8：依赖注入实现解耦

将以下紧耦合代码改写为使用依赖注入的松耦合版本，并演示切换不同通知方式。

```python
class OrderService:
    def place_order(self, order):
        # 紧耦合：直接依赖 EmailNotifier
        notifier = EmailNotifier()
        notifier.send(f"订单 {order} 已创建")
```

**答案：**
```python
class EmailNotifier:
    def send(self, message):
        print(f"[邮件] {message}")

class SMSNotifier:
    def send(self, message):
        print(f"[短信] {message}")

class OrderService:
    def __init__(self, notifier):
        self._notifier = notifier

    def place_order(self, order):
        self._notifier.send(f"订单 {order} 已创建")

# 使用邮件通知
email_svc = OrderService(EmailNotifier())
email_svc.place_order("ORD-001")
# 输出: [邮件] 订单 ORD-001 已创建

# 切换到短信通知——无需修改 OrderService 代码
sms_svc = OrderService(SMSNotifier())
sms_svc.place_order("ORD-002")
# 输出: [短信] 订单 ORD-002 已创建
```

## 🤔 常见误区

### 误区 1：「函数内修改变量一定会影响外部」

**事实**：这取决于对象的可变性。不可变对象（int、str、tuple）在函数内的赋值操作会创建新的局部变量，不会影响外部；只有对可变对象（list、dict）的原地修改（如 `.append()`）才会影响外部。理解「对象引用传递」机制是关键。

### 误区 2：「默认参数每次调用都会重新求值」

**事实**：默认参数在**函数定义时**只计算一次，之后每次调用都复用同一个对象。这就是为什么不能用 `[]` 或 `{}` 作为默认参数的原因——所有调用会共享同一个可变对象，导致状态累积。

### 误区 3：「Lambda 可以替代所有普通函数」

**事实**：Lambda 仅限于单行表达式，无法包含语句（如赋值、循环、异常处理）。滥用 Lambda 会使代码难以阅读和调试。如果逻辑需要多行，使用 `def` 定义命名函数是更明智的选择。

### 误区 4：「在函数内给变量赋值就是修改全局变量」

**事实**：在 Python 中，函数内的赋值操作默认创建**局部变量**。如果想修改全局变量，必须显式使用 `global` 关键字声明；修改外层函数的变量则需使用 `nonlocal`。忘记声明会导致 `UnboundLocalError` 或逻辑错误。

### 误区 5：「导入模块时用 `from module import *` 最方便」

**事实**：虽然省事，但这种方式会污染当前命名空间，容易造成命名冲突（如覆盖内置函数 `list`、`open`），且让代码的可追溯性变差。推荐使用 `import module` 或 `from module import specific_func`。
