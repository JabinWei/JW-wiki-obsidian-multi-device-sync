## 📚 核心内容

### 什么是反射？

**反射（Reflection）** 是指程序在**运行时**能够检查、访问和修改自身状态或行为的能力。简单说，就是在代码中"看"代码本身 —— 通过字符串名称来操作对象，而不是写死的 `.` 调用。

**没有反射，你只能这样做**（属性名写死在代码里）：
```python
obj.name        # 必须提前知道属性名叫 "name"
obj.run()       # 必须提前知道方法名叫 "run"
```

**有了反射，你可以这样做**（属性名可以是变量）：
```python
attr_name = "name"           # 属性名来自配置文件、用户输入、数据库...
method_name = "run"          # 方法名也可以在运行时决定

getattr(obj, attr_name)      # 等价于 obj.name
getattr(obj, method_name)()  # 等价于 obj.run()
```

**核心场景**：当你不知道属性/方法名是什么，需要根据字符串来动态访问时，就必须用到反射。

### 类型检查：`type()`、`isinstance()`、`issubclass()`

#### `type()` — 获取对象的类型

```python
print(type(42))           # <class 'int'>
print(type("hello"))      # <class 'str'>
print(type([1, 2, 3]))   # <class 'list'>
print(type(print))        # <class 'builtin_function_or_method'>
```

`type()` 返回的是**类本身**，可以用来做类型比较：

```python
x = 42
print(type(x) == int)     # True
print(type(x) == str)     # False
```

#### `isinstance()` — 判断对象是否为某类型（推荐，支持继承）

```python
class Animal:
    pass

class Dog(Animal):
    pass

d = Dog()

print(isinstance(d, Dog))      # True
print(isinstance(d, Animal))   # True  ← 子类实例也属于父类类型！
print(isinstance(d, object))   # True  ← 一切皆 object

# 也可以检查多种类型
print(isinstance(42, (int, float, complex)))  # True
```

> **`type()` vs `isinstance()`**：`type()` 只检查**精确类型**，不认继承；`isinstance()` 认继承关系。判断类型时默认用 `isinstance()`。

```python
d = Dog()
print(type(d) == Animal)       # False ← type 不认子类
print(isinstance(d, Animal))   # True  ← isinstance 认子类
```

#### `issubclass()` — 判断类是否为另一个类的子类

```python
class Animal: pass
class Dog(Animal): pass
class Cat(Animal): pass

print(issubclass(Dog, Animal))    # True
print(issubclass(Cat, Animal))    # True
print(issubclass(Dog, Cat))       # False
print(issubclass(Animal, Animal)) # True  ← 类是自身的子类
print(issubclass(bool, int))      # True  ← bool 是 int 的子类
```

### 对象探查：`dir()` 和 `vars()`

#### `dir()` — 列出所有属性和方法

```python
class Dog:
    species = "犬科"
    def __init__(self, name):
        self.name = name
    def bark(self):
        return "汪汪"

d = Dog("旺财")
print(dir(d))
# ['__class__', '__delattr__', '__dict__', ..., 'bark', 'name', 'species']
```

`dir()` 返回一个**排序后的字符串列表**，包含该对象的所有属性和方法（包括继承的、内置的）。不传参数时返回当前作用域的名字列表。

#### `vars()` — 查看 `__dict__`

```python
class Dog:
    species = "犬科"
    def __init__(self, name, age):
        self.name = name
        self.age = age

d = Dog("旺财", 3)

print(vars(d))
# {'name': '旺财', 'age': 3}   ← 只有实例属性，没有方法和类属性

print(vars(Dog))
# {'__module__': '__main__', 'species': '犬科', '__init__': ..., ...}  ← 类的字典
```

| | `dir()` | `vars()` |
|---|---|---|
| **返回内容** | 所有属性/方法名列表（含继承和内置） | `__dict__` 字典（只有自有属性） |
| **返回类型** | `list[str]` | `dict` |
| **无参数时** | 当前作用域的名字列表 | 等价于 `locals()` |
| **用途** | 探索对象有什么 | 查看对象的属性键值对 |

### 动态属性操作：`hasattr()`、`getattr()`、`setattr()`、`delattr()`

这是反射中最核心的四个函数，让你通过**字符串**来操作对象的属性。

#### `hasattr(obj, name)` — 检查是否有该属性

```python
class Person:
    def __init__(self, name):
        self.name = name

p = Person("张三")
print(hasattr(p, "name"))     # True
print(hasattr(p, "age"))      # False
print(hasattr(p, "__str__"))  # True  ← 从 object 继承的也算

# 典型用法：安全地访问不确定是否存在的属性
if hasattr(p, "age"):
    print(p.age)
else:
    print("对象没有 age 属性")
```

#### `getattr(obj, name, default)` — 获取属性值

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("张三", 25)

print(getattr(p, "name"))        # "张三"  ← 等价于 p.name
print(getattr(p, "age"))         # 25      ← 等价于 p.age

# 第三个参数：默认值（属性不存在时返回，不抛异常）
print(getattr(p, "address", "未知"))  # "未知"
print(getattr(p, "address"))          # AttributeError ← 没默认值就报错
```

#### `setattr(obj, name, value)` — 设置属性值

```python
class Person:
    def __init__(self, name):
        self.name = name

p = Person("张三")

setattr(p, "name", "李四")     # 等价于 p.name = "李四"
setattr(p, "age", 30)           # 等价于 p.age = 30（不存在的属性也能动态添加）

print(p.name, p.age)  # 李四 30
```

#### `delattr(obj, name)` — 删除属性

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("张三", 25)
delattr(p, "age")              # 等价于 del p.age
print(hasattr(p, "age"))       # False
```

#### 四个函数的对照总结

```python
class Dog:
    def __init__(self, name):
        self.name = name

d = Dog("旺财")

# 四件套：属性名的字符串 → 操作对象
hasattr(d, "name")                   # True       — 有吗？
getattr(d, "name")                   # "旺财"     — 拿来看看
getattr(d, "color", "未设置")        # "未设置"   — 没有就取默认
setattr(d, "color", "黄色")          # d.color = "黄色"
delattr(d, "name")                   # del d.name

# 这些都是等价的：
# d.name            ←→ getattr(d, "name")
# d.name = "xxx"    ←→ setattr(d, "name", "xxx")
# del d.name        ←→ delattr(d, "name")
```

### 动态方法调用

反射最强大的应用：通过字符串调用方法。

```python
class Calculator:
    def add(self, a, b):
        return a + b

    def subtract(self, a, b):
        return a - b

    def multiply(self, a, b):
        return a * b

# 假设方法名来自用户输入
calc = Calculator()
method_name = "add"             # 这可能是 input() 读到的

# getattr 拿到方法对象，再调用
func = getattr(calc, method_name)
result = func(3, 5)
print(result)  # 8

# 也可以一步到位
print(getattr(calc, "multiply")(4, 5))  # 20
```

**带安全检查的动态调用**：

```python
def safe_invoke(obj, method_name, *args, **kwargs):
    """安全地调用方法，不存在时给出友好提示"""
    if not hasattr(obj, method_name):
        print(f"对象没有 '{method_name}' 方法")
        return None
    func = getattr(obj, method_name)
    if not callable(func):
        print(f"'{method_name}' 不是方法，不能调用")
        return None
    return func(*args, **kwargs)

safe_invoke(calc, "add", 1, 2)       # 3
safe_invoke(calc, "divide", 6, 2)    # 对象没有 'divide' 方法
```

### `callable()` — 判断是否可调用

```python
def func():
    return "hello"

class Dog:
    def bark(self):
        return "汪汪"

d = Dog()

print(callable(func))      # True   — 函数
print(callable(d.bark))    # True   — 方法
print(callable(Dog))       # True   — 类本身也可调用（用于创建实例）
print(callable("hello"))   # False  — 字符串不可调用
print(callable(42))        # False
print(callable(print))     # True
```

### `__dict__` — 对象的属性字典

Python 中，对象的属性本质上存储在一个字典里，这个字典就是 `__dict__`。

```python
class Dog:
    species = "犬科"       # 类属性，存在 Dog.__dict__

    def __init__(self, name, age):
        self.name = name   # 实例属性，存在 实例.__dict__
        self.age = age

    def bark(self):
        return "汪汪"

d = Dog("旺财", 3)

# 实例的 __dict__：只有实例属性
print(d.__dict__)
# {'name': '旺财', 'age': 3}

# 类的 __dict__：类属性、方法都在这里
print(Dog.__dict__)
# {'__module__': '__main__', 'species': '犬科', '__init__': <function...>, 'bark': <function...>, ...}

# vars() 本质上就是返回 __dict__
print(vars(d) == d.__dict__)   # True
```

> **读取属性的底层逻辑**：当你写 `d.name` 时，Python 先到 `d.__dict__` 找，找不到再去 `type(d).__dict__`（即 `Dog.__dict__`）找，再往上到父类的 `__dict__` 找，一直找到 `object`。这就是属性查找的完整链条。

### 反射的实际应用场景

#### 场景 1：简易路由分发

```python
class Dispatcher:
    def handle_login(self):
        return "处理登录..."

    def handle_logout(self):
        return "处理退出..."

    def handle_register(self):
        return "处理注册..."

    def dispatch(self, action):
        """根据 action 字符串，动态调用对应方法"""
        method_name = f"handle_{action}"
        if hasattr(self, method_name):
            func = getattr(self, method_name)
            if callable(func):
                return func()
        return f"未知操作: {action}"

d = Dispatcher()
print(d.dispatch("login"))     # 处理登录...
print(d.dispatch("logout"))    # 处理退出...
print(d.dispatch("delete"))    # 未知操作: delete
```

#### 场景 2：对象序列化

```python
class User:
    def __init__(self, name, age, email):
        self.name = name
        self.email = email
        self.age = age

    def to_dict(self):
        """自动收集所有实例属性，导出为字典"""
        return {k: v for k, v in vars(self).items() if not k.startswith("_")}

    @classmethod
    def from_dict(cls, data):
        """从字典重建对象，跳过不存在的属性"""
        obj = cls.__new__(cls)  # 不调 __init__，直接创建裸对象
        for k, v in data.items():
            setattr(obj, k, v)      # ← 反射：字符串 → 属性赋值
        return obj

u = User("张三", 25, "zhangsan@example.com")
d = u.to_dict()
print(d)  # {'name': '张三', 'age': 25, 'email': 'zhangsan@example.com'}

u2 = User.from_dict({"name": "李四", "age": 30, "email": "lisi@example.com"})
print(u2.name, u2.age)  # 李四 30
```

#### 场景 3：配置驱动初始化

```python
class AppConfig:
    defaults = {"host": "localhost", "port": 8080, "debug": False}

    def __init__(self, **overrides):
        # 先设默认值
        for k, v in self.defaults.items():
            setattr(self, k, v)
        # 再用传入值覆盖
        for k, v in overrides.items():
            if hasattr(self, k):
                setattr(self, k, v)

config = AppConfig(port=9000, debug=True)
print(config.host)   # localhost  ← 来自默认值
print(config.port)   # 9000      ← 被覆盖
print(config.debug)  # True      ← 被覆盖
```

#### 场景 4：微型框架 — 自动发现与注册

这个例子展示了反射的核心价值：**框架不认识你的类，但它认识你遵守的约定**。

```python
import inspect
import sys

# ========= 框架部分（框架开发者写） =========

class MiniFramework:
    """微型框架：自动发现并管理所有 *Service 类"""

    def __init__(self):
        self.services = {}

    def scan(self, module):
        """扫描模块中所有类，自动注册符合约定的"""
        for name, cls in inspect.getmembers(module, inspect.isclass):
            if name.endswith("Service") and name != "BaseService":
                instance = cls()
                self.services[name] = instance
                print(f"[框架] 自动发现并注册: {name}")

    def run_all(self):
        """对每个注册的 Service 调用 execute 方法"""
        for name, svc in self.services.items():
            result = getattr(svc, "execute")()  # 反射调用，约定每个 Service 都有 execute
            print(f"[框架] {name}.execute() → {result}")


# ========= 用户代码（业务开发者写） =========

class UserService:
    def execute(self):
        return "创建用户成功"

class OrderService:
    def execute(self):
        return "下单成功"

class PaymentService:
    def execute(self):
        return "支付到账"

class SomeUtil:  # 不符合约定，不会被框架发现
    def do_stuff(self):
        pass


# ========= 运行 =========

fw = MiniFramework()
# 通过 sys.modules[__name__] 获取当前模块对象，传给 scan 扫描
fw.scan(sys.modules[__name__])

fw.run_all()
```

输出：

```
[框架] 自动发现并注册: UserService
[框架] 自动发现并注册: OrderService
[框架] 自动发现并注册: PaymentService
[框架] UserService.execute() → 创建用户成功
[框架] OrderService.execute() → 下单成功
[框架] PaymentService.execute() → 支付到账
```

**关键点**：

- 框架**不认识** `UserService`、`OrderService`，但它认识「以 `Service` 结尾 → 有 `execute` 方法」这个**模式**
- 新增一个 `ReportService` 时，框架一行代码都不用改，自动发现
- 不符合约定的 `SomeUtil` 被无视，但如果它也想被管理，就必须遵循约定
- 这就是「**约定大于配置**」的核心：约定给了框架一个「**无限的发现范围**」，而不是有限的已知名单

> **反射 + 约定 = 框架认识一类事物（所有符合模式的东西）**
> **直接指定 = 框架只认识一个事物（你手动注册的那个）**

### 练习 4：动态方法调用 — 简单计算器

实现一个 `Calculator` 类，用反射完成 `calculate(operation, a, b)` 方法：

```python
class Calculator:
    def add(self, a, b):
        return a + b

    def subtract(self, a, b):
        return a - b

    def multiply(self, a, b):
        return a * b

    def divide(self, a, b):
        if b == 0:
            return "除数不能为0"
        return a / b

    def calculate(self, operation, a, b):
        """根据 operation 字符串调用对应方法，不存在的方法返回友好提示"""
        # 你的代码
        pass

# 测试
calc = Calculator()
print(calc.calculate("add", 10, 5))        # 15
print(calc.calculate("divide", 10, 0))     # 除数不能为0
print(calc.calculate("power", 2, 3))       # 不支持的操作: power
```

**答案：**
```python
def calculate(self, operation, a, b):
    if hasattr(self, operation):
        func = getattr(self, operation)
        if callable(func):
            return func(a, b)
    return f"不支持的操作: {operation}"
```

## 🤔 常见误区

### 误区 1：混淆 `type()` 和 `isinstance()`

```python
class Animal: pass
class Dog(Animal): pass

d = Dog()
print(type(d) == Animal)      # False  ← 很多人以为会是 True
print(isinstance(d, Animal))  # True
```

**事实**：`type()` 只返回最精确的类型，不考虑继承链。判断"是不是某种类型"请用 `isinstance()`。

### 误区 2：忘记 `getattr` 的默认值参数

```python
obj = SomeClass()
try:
    value = obj.some_attr       # 万一没有就炸了
except AttributeError:
    value = None
```

**更好的写法**：`value = getattr(obj, "some_attr", None)`，一行搞定，无需 try/except。

### 误区 3：滥用反射导致代码难以维护

反射很强大，但也有代价：IDE 无法自动补全，重构时改名容易漏改，出问题时调用链不直观。

**原则**：能直接访问就不要用反射。只在**属性名本身是变量**时才用 —— 比如属性名来自配置文件、用户输入、数据库字段等动态来源。

### 误区 4：认为 `vars()` 返回包括方法和类属性

```python
class Dog:
    species = "犬科"
    def __init__(self, name):
        self.name = name

d = Dog("旺财")
print(vars(d))  # {'name': '旺财'}  ← 只有实例属性！
```

**事实**：`vars(obj)` 等价于 `obj.__dict__`，只返回实例自身的属性字典。类属性和方法在 `type(obj).__dict__` 里。
