## 📚 核心内容

### Python变量与常量

在 Python 的世界里，理解变量的本质是迈向 Pythonic 代码的第一步。Python 的变量机制与传统编程语言有着显著的不同，这种独特的设计赋予了它极高的灵活性。

#### 1. 变量

在 Python 中，变量更像是一个**指向对象的标签**，而不是一个存储数据的盒子。理解这一点是区分 `==`（值比较）和 `is`（身份比较）的关键基础，详见 [[PL-PY-001-is与==的区别]]。这种「引用模型」也直接决定了对象复制时的行为——浅拷贝只复制标签，深拷贝才复制整个对象图，详见 [[PL-004-浅拷贝与深拷贝]]。

- **动态类型 (Dynamic Typing)**
  Python 不需要在声明时指定类型，解释器会根据赋值自动推断。同一个变量名可以被赋予不同类型的数据。

  ```python
  message = "Hello Python world!" # message 指向一个字符串对象
  print(type(message)) # <class 'str'>
  message = 1 # message 现在指向一个整数对象
  print(type(message)) # <class 'int'>
  ```

  > **💡 注**：这种灵活性虽然方便，但也意味着你需要时刻关注变量当前的类型，避免类型错误。关于动态类型与强类型的深入讨论，详见 [[PL-001-编程语言的变量与类型系统]] 以及 [[PY001-简介#Python 到底是"强类型"还是"弱类型"？]]。

- **多重赋值**
  Python 支持一行代码给多个变量赋值，语法非常简洁：

  ```python
  x, y, z = 0, "str", True
  ```

- **命名规范 (PEP 8)**

  - **规则**：只能包含字母、数字和下划线，且不能以数字打头。
  - **大小写敏感**：`age`、`Age` 和 `AGE` 是三个完全不同的变量。
  - **风格**：推荐使用 **`snake_case`**（小写字母加下划线）。尽量以变量实际含义命名。
  - **禁忌**：不要使用关键字（如 `if`、`for`）。

#### 2. 常量

虽然 Python 没有语法层面的 `const`，但我们有约定俗成的规范。

| 实现方式                | 描述                             | 代码示例                                                     |
| ----------------------- | -------------------------------- | ------------------------------------------------------------ |
| **约定俗成**            | 全大写命名，依赖程序员自觉       | `PI = 3.14159`                                               |
| **Final (Python 3.8+)** | 配合类型检查工具，在编码阶段报错 | `from typing import Final` `DB_HOST: Final = "localhost"`    |
| **枚举 (Enum)**         | 强制不可变，运行时报错           | `from enum import Enum` `class Constants(Enum):` `MAX_USERS = 100` |

## 🧪 练习 / 验证

### 练习 1：变量类型追踪

阅读以下代码，写出每一行 `print` 的输出结果。

```python
data = 42
print(type(data))

data = "forty-two"
print(type(data))

data = [4, 2]
print(type(data))
```

**答案：**
```
<class 'int'>
<class 'str'>
<class 'list'>
```
解释：Python 是动态类型语言，同一个变量 `data` 可以先后指向不同类型的对象。

### 练习 2：多重赋值与变量交换

利用 Python 的多重赋值特性，在不使用临时变量的情况下交换 `a` 和 `b` 的值。补全代码并写出输出。

```python
a = 100
b = 200

# 你的代码：交换 a 和 b 的值

print(a, b)
```

**答案：**

```python
a = 100
b = 200

a, b = b, a  # 一行交换

print(a, b)  # 输出: 200 100
```

### 练习 3：变量命名合法性判断

判断以下变量名是否合法，对不合法的说明原因。

| 变量名 | 是否合法 | 原因 |
|--------|---------|------|
| `my_var` | ？ | ？ |
| `2nd_place` | ？ | ？ |
| `_hidden` | ？ | ？ |
| `class` | ？ | ？ |
| `user-name` | ？ | ？ |
| `MAX_SIZE` | ？ | ？ |

**答案：**

| 变量名 | 是否合法 | 原因 |
|--------|---------|------|
| `my_var` | 合法 | 小写字母 + 下划线，符合 snake_case |
| `2nd_place` | 不合法 | 不能以数字开头 |
| `_hidden` | 合法 | 下划线开头是允许的（习惯上表示私有变量） |
| `class` | 不合法 | `class` 是 Python 关键字 |
| `user-name` | 不合法 | 连字符 `-` 在 Python 中被解释为减号 |
| `MAX_SIZE` | 合法 | 全大写 + 下划线，符合常量命名惯例 |

## 🤔 常见误区

### 误区 1：「变量是存储数据的盒子」

**事实**：在 Python 中，变量是**指向对象的标签/引用**，而不是存储数据的容器。理解这一点对于掌握 Python 的内存模型至关重要。当你执行 `a = [1, 2, 3]` 时，`a` 并不是"装着列表的盒子"，而是贴在列表对象上的一个标签。如果随后执行 `b = a`，`b` 和 `a` 指向同一个列表对象，修改 `b` 会影响 `a`。

```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)  # [1, 2, 3, 4] — a 也被影响了！
```

### 误区 2：「常量在 Python 中是不能修改的」

**事实**：Python 语法层面并没有真正的常量机制。全大写变量名（如 `PI = 3.14`）只是一个**约定**，告诉其他开发者"请不要修改这个值"，但 Python 解释器并不会阻止你重新赋值。如果需要强制不可变性，应使用 `Final`（Python 3.8+，配合类型检查器）或 `Enum`。
